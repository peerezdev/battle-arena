# Mobile UI pass (main flows) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the main user-facing flows (Gacha, the two card-detail modals, lobby LiveBattles, the shell topbar, Hub/Profile padding) usable on a ~360px phone, without changing the existing wide layouts.

**Architecture:** Inline styles switched by the existing `useIsWide(query)` hook (and `auto-fill` grids where no hook is needed). No new CSS, no new deps. Every change is gated behind a narrow branch so desktop layouts stay byte-for-byte identical.

**Tech Stack:** React + TS + vitest.

## Global Constraints

- Use the existing `useIsWide(query: string): boolean` from `src/ui/useIsWide.ts`; or `repeat(auto-fill, minmax(...))`. No new CSS files / deps. Reuse `COLORS`/`FONTS`.
- **Wide (desktop) layouts unchanged** — only the narrow branch is new.
- Breakpoints (verbatim): Gacha stack `(min-width: 880px)`; pool tiles `(min-width: 560px)`; detail modals `(min-width: 620px)`; inventory grading grid `(min-width: 380px)`; topbar balance `(min-width: 480px)`.
- Presentational only — no new unit tests; verify with `npx tsc --noEmit` + `npm run build`; keep the vitest suite green.
