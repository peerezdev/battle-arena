- [ ] **Step 1: Reescribir `src/wallet/useWallet.ts`** manteniendo EXACTA la interfaz `WalletApi`:
```ts
export interface WalletApi {
  publicKey: PublicKey | null
  isConnected: boolean
  connect: () => void
  signAndSendTransaction: (ixs: TransactionInstruction[]) => Promise<string>
  signMessage: (message: Uint8Array) => Promise<Uint8Array>
  signTransactionBase64: (txBase64: string) => Promise<string>
}
```
Implementación sobre Privy (NO Reown):
- `import { usePrivy } from '@privy-io/react-auth'` y `import { useWallets } from '@privy-io/react-auth/solana'`.
- **Confirma en los tipos instalados** (`node_modules/@privy-io/react-auth/dist/dts/solana.d.ts` y `@privy-io/js-sdk-core`) los métodos de la wallet Solana conectada (`ConnectedStandardSolanaWallet`): nombres exactos para firmar mensaje, firmar tx, y firmar-y-enviar. NO inventes.
- `const { authenticated, login } = usePrivy()`; `const { wallets } = useWallets()`; `const wallet = wallets[0] ?? null`.
- `publicKey`: `wallet ? new PublicKey(wallet.address) : null`.
- `isConnected`: `authenticated && !!wallet`.
- `connect`: `() => login()`.
- `signMessage(message)`: usar el método de la wallet de Privy (devuelve la firma `Uint8Array`; adapta si Privy devuelve `{signature}`).
- `signAndSendTransaction(ixs)`: `new Connection(config.rpcUrl, 'confirmed')`; `getLatestBlockhash`; `const tx = new Transaction(); tx.feePayer = publicKey; tx.recentBlockhash = blockhash; tx.add(...ixs)`; firmar+enviar con el método de la wallet de Privy (o firmar con la wallet y `connection.sendRawTransaction`); `confirmTransaction`; devolver la signature string.
- `signTransactionBase64(txBase64)`: `Transaction.from(Buffer.from(txBase64,'base64'))`; firmar (sin enviar) con la wallet de Privy; `Buffer.from(signed.serialize()).toString('base64')`.
- Mantén guards: si `!isConnected || !publicKey` lanza `new Error('Wallet not connected')`; si no hay wallet de Privy disponible lanza error claro.
- Importa `config` de `../onchain/config`. Quita TODO import de `@reown/*`.
