# Privy B3 (identidad on-chain vía identity token) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Los endpoints on-chain del backend autentican con el identity token de Privy (→ embedded Solana wallet) en vez de la firma de wallet; el frontend manda ese token.

**Architecture:** `PrivyVerifier` gana `embedded_solana_wallet(token)`; un dependency `current_user` reemplaza `current_wallet`; se quitan `/auth/nonce`+`/auth/verify` y la firma. El frontend activa identity tokens, los obtiene de Privy y los manda como bearer; `ConnectScreen` se simplifica.

**Tech Stack:** FastAPI + pyjwt[crypto]; React + @privy-io/react-auth. Backend tests `cd backend && .venv/bin/python -m pytest -q`; frontend `npx tsc --noEmit && npx vitest run && npm run build`.

**Spec:** `docs/superpowers/specs/2026-06-10-privy-b3-design.md`.

- [ ] **Step 3:** Implementa en `backend/app/privy.py` el método (junto a `verify`):
```python
    def embedded_solana_wallet(self, token: str) -> str:
        """Verifica un identity token de Privy y devuelve la dirección de la
        embedded Solana wallet de linked_accounts."""
        claims = self.verify(token)  # mismo JWKS/aud/iss/exp que el access token
        raw = claims.get("linked_accounts")
        try:
            accounts = json.loads(raw) if isinstance(raw, str) else (raw or [])
        except (TypeError, ValueError):
            raise PrivyAuthError("linked_accounts ilegible")
        for acc in accounts:
            if (acc.get("type") == "wallet" and acc.get("chain_type") == "solana"
                    and acc.get("connector_type") == "embedded" and acc.get("address")):
                return acc["address"]
        raise PrivyAuthError("sin embedded Solana wallet")
```
